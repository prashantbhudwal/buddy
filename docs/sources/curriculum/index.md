# Curriculum Sources

This folder is the evidence base for Buddy's curriculum model.

- Raw PDF conversions live in `docs/sources/curriculum/raw/`.
- `principles.md` distills the source-backed rules Buddy actually implements.
- `crosswalk.md` maps those rules into the intent docs, spec, prompts, and runtime.

## Source Inventory

| Source | Raw text | Why it matters |
| --- | --- | --- |
| `Carl Wieman Science Education Initiative Request for Proposals .pdf` | `raw/carl-wieman-science-education-initiative-request-for-proposals.txt` | Sets the CWSEI framing: measurable learning goals, objective assessment, teaching methods aimed at those goals, and sustained optimization. |
| `COURSE SCALE LEARNING GOALS.pdf` | `raw/course-scale-learning-goals.txt` | Concrete examples of course-scale goals that emphasize communication, organized knowledge, visualization, and transfer beyond routine procedures. |
| `CourseTransformationGuide_CWSEI_CU-SEI.pdf` | `raw/coursetransformationguide-cwsei-cu-sei.txt` | Primary umbrella source for course transformation, learning principles, alignment, feedback, and active learning. |
| `CPSC_Learning_Goals.pdf` | `raw/cpsc-learning-goals.txt` | Computer-science-specific examples showing how goals, topic grids, and assessment alignment can be expressed in a programming curriculum. |
| `Creating and Using Effective Learning Goals.pdf` | `raw/creating-and-using-effective-learning-goals.txt` | Core source for the "At the end of this course/topic, students will be able to..." goal pattern, explicit verbs, and topic/course goal layering. |
| `Creating good homework problems (and grading them).pdf` | `raw/creating-good-homework-problems-and-grading-them.txt` | Main source for practice-first pedagogy, expert-thinking components a-j, why-it-matters context, and feedback that learners must act on. |
| `CWSEI Teaching Practices Inventory.pdf` | `raw/cwsei-teaching-practices-inventory.txt` | Checklist-style source for competencies, worked examples, practice, pre-class preparation, feedback loops, and learner agency. |
| `Good Examples of Learning Goals at UBC and CU.pdf` | `raw/good-examples-of-learning-goals-at-ubc-and-cu.txt` | Broad corpus of high-quality goals and prerequisite/alignment examples across domains, including CS. |
| `How People Learn--Implications for teac.pdf` | `raw/how-people-learn-implications-for-teac.txt` | Source for the deeper learning principles Buddy uses: prior thinking, motivation, cognitive load, effortful practice, and meaningful transfer. |
| `How to Develop Learning Goals for an Established Course- The Computer Science Model1.pdf` | `raw/how-to-develop-learning-goals-for-an-established-course-the-computer-science-model1.txt` | CS-specific process for deriving goals from assessments and teaching artifacts, plus course/topic goal structuring. |
| `Learning Goals for UBC PHYS 250, Introduction to Modern Physics, Summer 2009 .pdf` | `raw/learning-goals-for-ubc-phys-250-introduction-to-modern-physics-summer-2009.txt` | Example of goals that explicitly include motivation, evidence, and "learn how to learn" expectations. |
| `Role Play- A Glimpse into the Process of Creating Learning Goals (developed Oct 2008, Beth Simon and Steve Wolfman for the UBC CWSEI) .pdf` | `raw/role-play-a-glimpse-into-the-process-of-creating-learning-goals-developed-oct-2008-beth-simon-and-steve-wolfman-for-the-ubc-cwsei.txt` | Shows the actual conversational refinement process from vague exam facts to durable learning goals. |
| `Tracking Changing Learning Goals (long version) .pdf` | `raw/tracking-changing-learning-goals-long-version.txt` | Useful for iterative goal refinement, explicit alignment to assessments, and course revision based on observed learner struggle. |
| `What key elements must we preserve to sustain the use of evidence-based teaching methods and drive further improvement of science education at UBC?.pdf` | `raw/what-key-elements-must-we-preserve-to-sustain-the-use-of-evidence-based-teaching-methods-and-drive-further-improvement-of-science-education-at-ubc.txt` | Supports the emphasis on local expertise, evidence-based teaching, and organizational support for translating principles into practice. |

## How to use these sources

- Use `principles.md` when deciding what Buddy should do pedagogically.
- Use `crosswalk.md` when deciding where a principle lives in the product or runtime.
- Use the raw texts when the curated summaries are not enough and a source needs to be checked directly.
