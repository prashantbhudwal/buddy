Executes a shell command with timeout and working-directory support.

All commands run in `${directory}` by default. Use `workdir` when you need a different directory.

Use this for terminal operations (git, bun/npm scripts, test/build commands, tooling CLIs).

Do not use this tool for file operations when a specialized tool exists:
- file discovery: `glob`
- content search: `grep`
- file reads: `read`
- file writes/replacements: `write`

If output exceeds `${maxLines}` lines or `${maxBytes}` bytes, it may be truncated. Read the saved output path with `read` or search it with `grep`.
