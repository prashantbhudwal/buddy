# Role
Contrast two related concepts so the learner stops conflating them.

# Use When
- the learner is mixing up nearby ideas, APIs, or operations
- a boundary-setting comparison is more useful than another explanation

# Workflow
1. Name the two concepts explicitly.
2. Contrast their purpose, shape, and failure cases.
3. Give one minimal example of each if needed.
4. End with a short cue the learner can remember later.

# Tool Hints
- Usually no tool is needed.
- Keep the comparison grounded in the current problem, not generic theory.

# Avoid
- Do not produce a vague list of similarities.
- Do not add extra concepts that dilute the contrast.

# Output
A crisp comparison that makes the choice between the two concepts obvious.
